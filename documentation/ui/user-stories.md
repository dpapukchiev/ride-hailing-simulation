## UI

- Story: As a user, I can visualize riders, drivers, and trips on a map with
  state-based colors.
  Status: Done

- Story: As a user, I can control the simulation (start, pause, step, step 100, run to
  end, reset) and adjust scenario parameters before run.
  Status: Done

- Story: As a user, I can view time-series charts for active trips, waiting riders,
  idle drivers, cancellations, abandoned quotes, and completed/cancelled trips.
  Status: Done

- Story: As a user, I can view a detailed trip table with all trips showing
  timestamps, distances, and states sorted by last updated time.
  Status: Done

- Story: As a user, I can view fleet metrics including driver utilization,
  earnings distributions, and fatigue tracking.
  Status: Done

- Story: As a user, I can view run outcomes including conversion rates and timing
  distributions (time to match, time to pickup, trip duration) with percentiles.
  Status: Done

- Story: As a user, I can toggle driver stats (earnings/fatigue) display on the map.
  Status: Done

- Story: As a user, I can change the matching algorithm during simulation execution.
  Status: Done

- Story: As a user, I can configure simulation start time (year/month/day/hour/minute UTC)
  with a "Now" button to set it to current time.
  Status: Done

- Story: As a user, I can configure initial rider and driver counts separately from
  total counts.
  Status: Done

- Story: As a user, I can configure rider and driver spread windows separately.
  Status: Done

- Story: As a user, I can load and save scenario presets for quick iteration.
  Status: Backlog

- Story: As a user, I can see a Berlin street map rendered beneath the
  simulation agents on the UI, providing real-world geographic context for
  rider/driver positions and trip routes.
  Status: Backlog
